# Pattern Blueprint: Dijkstra / Shortest Path

## 1. System Design Mapping
* **Macro System Component:** Service Mesh Traffic Router (Envoy Routing Path Optimization)
* **How it leverages this DSA Pattern:**
  Rerouting RPC calls across microservices requires finding the path with minimum cumulative latency (network ping times). Containers act as vertices and connections represent edges weighted by ping responses. Dijkstra's algorithm finds the optimal route path.
* **Data Flow Architecture:**
  ```text
  Source Node A ──> [Query Shortest Paths via Latency Weights] ──> Select Minimum Weight Node ──> Route Traffic
  ```

## 2. High-Yield Performance Tricks (Java Specific)
* **Memory & Collection Strategy:**
  Avoid standard `PriorityQueue<Vertex>` wrappers. Use custom parallel heap arrays or primitive arrays tracking node distances and visitation.
* **Time/Space Constraints:**
  * Time Complexity: $O(E \log V)$ using binary heaps.
  * Space Complexity: $O(V)$ distance tracking storage.

## 3. The Core Structural Trick (Mental Model)
> Continually choose the unvisited node with the absolute smallest tentative distance, relax its neighbors' distances, and mark it visited.
