# Pattern Blueprint: Topological Sort

## 1. System Design Mapping
* **Macro System Component:** Distributed Task Scheduler / CI-CD DAG Orchestrator
* **How it leverages this DSA Pattern:**
  Workflow engines (e.g. Airflow, GitLab CI) coordinate jobs with complex dependency chains (e.g., Task B and C depend on Task A; Task D depends on B and C). Topological sorting determines the exact linear order of execution. Using Kahn's algorithm, we track task in-degrees, queue items with zero dependencies, and decrement neighbor counts as tasks complete.
* **Data Flow Architecture:**
  ```text
  Workflow Definition ──> [Calculate In-Degrees] ──> [Queue 0-In-Degree Tasks]
                                                             │
                                             ┌───────────────┴───────────────┐
                                             ▼                               ▼
                                       Execute Task             Cycle Detected (If tasks left 
                                             │                   but queue empty)
                              [Decrement Neighbors' In-Degree]
                                             │
                              [In-degree becomes 0? -> Queue it]
  ```

## 2. High-Yield Performance Tricks (Java Specific)
* **Memory & Collection Strategy:**
  Use flat `int[]` arrays to store the in-degree count of each task index. Implement Kahn's queue using a pre-allocated primitive circular queue `int[] queue` to avoid the heap allocation overhead of queue node containers.
* **Time/Space Constraints:**
  * Time Complexity: $O(V + E)$ to evaluate all tasks and dependencies.
  * Space Complexity: $O(V)$ in-degree array, $O(V)$ queue size.
* **Garbage Collection (GC) Impact:**
  Extremely low. Pre-allocating the in-degree array and queue ensures that task sorting can execute in rapid succession without generating object allocations.

## 3. The Core Structural Trick (Mental Model)
> Continually process nodes with an in-degree of 0 (no dependencies) and remove their outgoing edges. If nodes remain unprocessed but the queue is empty, a dependency cycle exists.
