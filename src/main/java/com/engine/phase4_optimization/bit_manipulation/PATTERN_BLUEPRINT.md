# Pattern Blueprint: Bit Manipulation & Math

## 1. System Design Mapping
* **Macro System Component:** High-Throughput Cache Filters (Bloom Filters) & Cardinality Estimators (HyperLogLog)
* **How it leverages this DSA Pattern:**
  Caching layers (e.g. Redis, Cloudflare filters) use bitwise structures to check database membership. Bloom filters hash keys to multiple bit offsets inside a bitwise mask. HyperLogLog estimates cardinalities by tracking leading binary zeros of hashed keys.
* **Data Flow Architecture:**
  ```text
  Key Search ──> [Hash Key to Bit Offsets] ──> [Bitwise AND check on long[] array] ──> Return Fast Miss/Likely Hit
  ```

## 2. High-Yield Performance Tricks (Java Specific)
* **Memory & Collection Strategy:**
  Do not use wrapper types or standard arrays. Use primitive `long[]` arrays to represent bit arrays, where each `long` holds 64 bits.
* **Bitwise Arithmetic:**
  * To check a bit: `(bitmap[index / 64] & (1L << (index % 64))) != 0`
  * Fast division/modulo using bitwise operators: `index >>> 6` (divide by 64) and `index & 63` (modulo 64).
  * Use JVM-intrinsic hardware acceleration: `Long.numberOfLeadingZeros()` and `Long.bitCount()`.
* **Time/Space Constraints:**
  * Time Complexity: $O(H)$ where $H$ is the number of hash functions (constant time).
  * Space Complexity: $O(B)$ where $B$ is bit array capacity.

## 3. The Core Structural Trick (Mental Model)
> Pack boolean states and metrics into the bits of primitive integers, resolving updates via register-level bitwise operations.
