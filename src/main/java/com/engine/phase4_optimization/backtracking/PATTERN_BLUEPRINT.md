# Pattern Blueprint: Backtracking

## 1. System Design Mapping
* **Macro System Component:** SQL Relational Query Execution Planner & Optimizer
* **How it leverages this DSA Pattern:**
  SQL query execution planners search for the lowest cost join tree. The engine traverses possible table join orders using backtracking. Cost estimators score sub-paths; if a partial path exceeds the current minimum cost threshold, the branch is pruned early, and the planner backtracks.
* **Data Flow Architecture:**
  ```text
  Tables Join Options ──> [Recursively Try Permutations] ──> Check Cost Bound 
                                                                    │
                                                ┌───────────────────┴───────────────────┐
                                                ▼                                       ▼
                                         Cost > Best Min                         Cost < Best Min
                                                │                                       │
                                          [Prune Branch]                          [Update Min & Keep Searching]
  ```

## 2. High-Yield Performance Tricks (Java Specific)
* **Memory & Collection Strategy:**
  Avoid allocating arrays or lists at each level of recursion. Maintain a single shared primitive array buffer (e.g. `int[] currentPermutation`) and index pointer, modifying and reverting changes in-place.
* **Time/Space Constraints:**
  * Time Complexity: $O(N!)$ or $O(2^N)$ in worst-case (mitigated by pruning).
  * Space Complexity: $O(N)$ recursion stack depth.

## 3. The Core Structural Trick (Mental Model)
> Systematically search choices recursively, updating a single global state tracker. Backtrack by reversing state changes immediately after recursive calls complete.
