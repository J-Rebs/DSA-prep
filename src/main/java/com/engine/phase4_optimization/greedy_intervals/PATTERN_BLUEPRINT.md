# Pattern Blueprint: Greedy & Intervals

## 1. System Design Mapping
* **Macro System Component:** Thread Pool Task Scheduler (SJF/EDF schedulers)
* **How it leverages this DSA Pattern:**
  High-concurrency job runners must schedule tasks that share resources without overlapping. The scheduler sorts task intervals by end times and greedily schedules tasks that finish first, maximizing resource utilization efficiency.
* **Data Flow Architecture:**
  ```text
  Raw Task Intervals ──> [Sort by Finish Time] ──> [Greedily Select Earliest Finisher] ──> Run Task
  ```

## 2. High-Yield Performance Tricks (Java Specific)
* **Memory & Collection Strategy:**
  Avoid allocating interval objects. Flatten intervals into two primitive arrays `int[] starts` and `int[] ends` and sort them using primitive Dual-Pivot Quicksort.
* **Time/Space Constraints:**
  * Time Complexity: $O(N \log N)$ (dominated by sorting).
  * Space Complexity: $O(1)$ auxiliary memory (if sorted in-place).

## 3. The Core Structural Trick (Mental Model)
> Make the locally optimal choice at each step (e.g. earliest end-time first) to achieve a globally optimal scheduling configuration.
