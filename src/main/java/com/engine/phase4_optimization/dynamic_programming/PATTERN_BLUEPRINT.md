# Pattern Blueprint: Dynamic Programming

## 1. System Design Mapping
* **Macro System Component:** Cloud Instance Task Load Balancer & Bin-Packing Optimizer
* **How it leverages this DSA Pattern:**
  Assigning cloud tasks (containers) to host instances to maximize resource density (CPU/RAM packing) is an NP-hard optimization problem. Using dynamic programming state tables, we compute subproblem allocations bottom-up, yielding optimal task layouts.
* **Data Flow Architecture:**
  ```text
  Containers to Allocate ──> [Evaluate Subproblem Weights/CPU/RAM capacities] ──> [DP State Table] ──> Optimal Placement
  ```

## 2. High-Yield Performance Tricks (Java Specific)
* **Memory & Collection Strategy:**
  Avoid nesting `Map` structures or using heavy boxed tables. Allocate pre-allocated flat 1D or 2D primitive arrays (`int[] dp`).
* **Space Compression:**
  For multi-dimensional DP where state updates only depend on the previous row, compress the 2D table into a 1D array to improve CPU cache-locality.
* **Time/Space Constraints:**
  * Time Complexity: $O(N \cdot W)$ where $N$ is items and $W$ is capacity.
  * Space Complexity: $O(W)$ with compressed 1D state arrays.

## 3. The Core Structural Trick (Mental Model)
> Solve overlapping subproblems in bottom-up order, storing solutions in a state table. Eliminate recursion overhead by converting top-down DFS memoization to iterative loops.
