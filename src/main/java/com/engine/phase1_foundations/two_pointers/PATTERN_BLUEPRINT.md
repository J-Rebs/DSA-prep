# Pattern Blueprint: Two Pointers

## 1. System Design Mapping
* **Macro System Component:** Real-time Stream Compaction & In-Place Log Deduplication
* **How it leverages this DSA Pattern:**
  High-throughput database engines or stream processors receive events in sorted chronological order. To save network bandwidth or storage before serialization, events must be deduplicated in-place. We use a slow-runner pointer `i` representing the last unique element, and a fast-runner pointer `j` scanning the rest of the stream.
* **Data Flow Architecture:**
  ```text
  Input Array:  [ E1, E1, E2, E3, E3, E4 ]
                  ▲   ▲
                  │   │
               slow  fast  --> elements equal? yes -> advance fast
               
  Input Array:  [ E1, E1, E2, E3, E3, E4 ]
                  ▲       ▲
                  │       │
               slow      fast  --> elements equal? no -> increment slow, write: array[slow] = array[fast]
  ```

## 2. High-Yield Performance Tricks (Java Specific)
* **Memory & Collection Strategy:**
  Avoid allocating auxiliary lists or maps to track unique items. Perform the modification in-place inside the incoming batch's byte buffer or primitive event array.
* **Time/Space Constraints:**
  * Time Complexity: $O(N)$ single pass scan.
  * Space Complexity: $O(1)$ auxiliary memory.
* **Garbage Collection (GC) Impact:**
  Zero allocation. Reusing the input array directly prevents object creation, keeping JVM memory footprint completely flat.

## 3. The Core Structural Trick (Mental Model)
> Use one write pointer (slow-runner) that only advances when a new, unique value is discovered by the scan pointer (fast-runner).
