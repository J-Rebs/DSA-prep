# Pattern Blueprint: Heaps & Priority Queue

## 1. System Design Mapping
* **Macro System Component:** Distributed Top-K Heavy Hitters Tracker (DDoS IP Tracker)
* **How it leverages this DSA Pattern:**
  We must track the top $K$ most frequent items (e.g., source IP addresses) in a stream. A frequency map (or Count-Min Sketch) tracks item counts. A Min-Heap of size $K$ maintains the current top $K$ elements. If an updated count exceeds the value at the root of the Min-Heap, the root is popped and the updated element is pushed into the heap.
* **Data Flow Architecture:**
  ```text
  Stream Event ──> [Frequency Map / CM Sketch] ──> Count Updated ──> [Check Min-Heap Root]
                                                                               │
                                                             ┌─────────────────┴─────────────────┐
                                                             ▼                                   ▼
                                                      Count <= Min Root                   Count > Min Root
                                                             │                                   │
                                                        [Do Nothing]                 [Pop Root & Insert Element]
  ```

## 2. High-Yield Performance Tricks (Java Specific)
* **Memory & Collection Strategy:**
  `java.util.PriorityQueue` works with boxed Objects and triggers continuous object creation and pointer chasing. For high-throughput scenarios, implement an inline, primitive binary heap using parallel arrays (e.g., `int[] ids` and `int[] counts`) to guarantee cache-locality and zero boxing.
* **Time/Space Constraints:**
  * Time Complexity: $O(\log K)$ per update.
  * Space Complexity: $O(K)$ heap size.
* **Garbage Collection (GC) Impact:**
  Completely avoids GC overhead by pre-allocating the heap structures, eliminating garbage collections under continuous ingestion.

## 3. The Core Structural Trick (Mental Model)
> Use a Min-Heap of size K. The root element acts as the gatekeeper; any item with a count greater than the root must replace the root to stay in the top-K.
