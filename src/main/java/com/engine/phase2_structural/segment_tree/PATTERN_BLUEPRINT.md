# Pattern Blueprint: Segment Tree / Fenwick Tree

## 1. System Design Mapping
* **Macro System Component:** Telemetry Metric Range Query Engine (TSDB aggregations)
* **How it leverages this DSA Pattern:**
  Databases require sliding aggregates (sums, minimums, maximums) over time slices. A segment tree precalculates aggregates over intervals, allowing any range query to be resolved in logarithmic time rather than scanning all raw values.
* **Data Flow Architecture:**
  ```text
  Data Point Update ──> [Point Update Segment Tree] ──> Logarithmic updates up to Root
  Range Query [T1, T2] ──> [Traverse Interval Nodes] ──> Combine results in O(log N)
  ```

## 2. High-Yield Performance Tricks (Java Specific)
* **Memory & Collection Strategy:**
  Do not use dynamic node objects. Store the segment tree inside a flat 1D array of size `4 * N`.
* **Array Navigation Math:**
  * Left child of index `i` is at `i << 1`.
  * Right child of index `i` is at `(i << 1) | 1`.
* **Time/Space Constraints:**
  * Query Time: $O(\log N)$
  * Update Time: $O(\log N)$
  * Space Complexity: $O(N)$ array size.

## 3. The Core Structural Trick (Mental Model)
> Divide interval metrics into halves and store aggregate stats. Any arbitrary range query can be reconstructed by merging at most $O(\log N)$ node aggregates.
